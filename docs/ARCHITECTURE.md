## 模块说明

### 1. USB 通信层

ESP32-S3 充当 USB OTG Host，手机是 USB Device。

**两条逻辑通道：**

| 通道 | 协议 | 方向 | 作用 |
|------|------|------|------|
| 音频流 | UAC 1.0 (USB Audio Class) | 手机 → KnobTone | 音频数据传输 |
| 控制指令 | HID (Human Interface Device) | KnobTone → 手机 | 音量、播放、切歌等媒体控制 |

> 💡 ESP32-S3 不处理音频数据本身——音频流透传给 DAC 芯片。ESP32 只负责 HID 控制和力反馈。

### 2. DAC 音频层

使用成熟的 USB DAC 芯片（如 CX31993），由手机直接通过 UAC 协议驱动。

**选型考虑：**
- **方案 A（推荐）**：成品 USB DAC 小尾巴拆机板。优点是 DAC 芯片 + 外围电路 + 3.5mm 座都在一块板上，直接接 USB 信号即可。
- **方案 B**：独立的 I2S DAC 模块（如 PCM5102）。需要 ESP32 接收 USB 音频、解码、再通过 I2S 输出。复杂度高，作为后续迭代。

### 3. 力反馈层

硬件链路：
```
AS5600 编码器 → I2C → ESP32-S3 → PWM → TMC6300 → BLDC 电机
```

软件链路：
```
旋钮位置读取 → SimpleFOC 闭环控制 → 力矩输出 → 触觉反馈
```

**四种反馈模式：**

| 模式 | Detent | 阻尼 | 边界反馈 |
|------|--------|------|---------|
| 音量 | 100 步/圈 | 中等 | 轻阻力 |
| 切歌 | 每首歌一步 | 中等 | 重阻力「咚」 |
| 快进 | 无 | 轻 | 无 |
| EQ | 每个预设一步 | 中等 | 轻阻力 |

### 4. 电源层

```
USB-C (手机端，5V)
        │
        ▼
  ┌──────────┐
  │ TP4056   │ ← 充电管理
  │ 充电     │
  └────┬─────┘
       │
       ▼
  ┌──────────┐
  │ 锂电池    │ ← 3.7V 500mAh
  │          │
  └────┬─────┘
       │
       ├──→ MT3608 (升压 5V) → DAC
       │
       └──→ LDO (降压 3.3V) → ESP32-S3, AS5600, TMC6300
```

## 软件栈

```
固件 (ESP32-S3 / PlatformIO)
├── main.cpp              ← 入口：初始化 + 主循环
├── knob.cpp / knob.h     ← 旋钮读取 + SimpleFOC 封装
├── modes.cpp / modes.h   ← 模式管理（音量/切歌/快进/EQ）
├── hid_report.cpp        ← USB HID 媒体键上报
├── haptics.cpp           ← 力反馈参数管理
└── config.h              ← 引脚定义 + 参数常量
```

## 开发阶段与生产阶段

| 组件 | Phase 1（原型） | Phase 3（PCB） |
|------|----------------|----------------|
| MCU | ESP32-S3-DevKitC | ESP32-S3-WROOM-1 模组 |
| 旋钮 | KY-040 编码器 | BLDC + AS5600 |
| DAC | CX31993 成品小尾巴 | CX31993 芯片 + 外围电路 |
| 连接 | 面包板 + 杜邦线 | 4 层 PCB |
| 外壳 | 无 | 3D 打印 / CNC |
