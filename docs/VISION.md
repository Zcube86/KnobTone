# KnobTone 产品愿景

## 一句话

**一个用物理旋钮 + 力反馈重新定义移动端 HiFi 交互的开源解码耳放。**

## 我们要解决什么问题

手机听歌有两个「打断」：

1. **操作打断**：调音量、切歌、快进——每次都要摸手机、点亮屏幕、看着操作
2. **体验打断**：触摸屏的滑动和点击没有触觉反馈，手指不知道「滑了多少」「选到哪了」

KnobTone 的答案是：**把所有操作交还给手指的触觉和肌肉记忆。**

## 产品定义

> KnobTone 不是「一个带旋钮的小尾巴」。  
> KnobTone 是「把触摸播放器重新设计成物理交互播放器的设备」。

### 产品形态

- **外形**：USB-C 解码耳放（小尾巴）形态，比普通小尾巴稍宽，侧面有一个突出的金属旋钮
- **连接**：USB-C 公头直插手机，另一端 3.5mm 耳机孔
- **旋钮位置**：轴平行于手机侧面，旋钮面垂直于手机屏幕——可以单手握住手机的同时用食指旋转
- **供电**：内置锂电池，不消耗手机电量

### 目标用户

- 喜欢盲操作、不想频繁掏手机的 HiFi 爱好者
- 对交互体验有要求的硬件 DIY 玩家
- 想做「不一样的音频产品」的独立开发者

## 设计原则

> **任何在手机上需要滑动的操作，都应该可以通过旋钮完成；任何需要点击确认的操作，都应该通过按压完成。**

这个原则贯穿整个产品的交互设计、硬件选型和固件架构。

## 不做的事情

- 不做屏幕（手机就是屏幕）
- 不做无线（第一代不走蓝牙，保证音质和低延迟）
- 不自己设计 DAC 电路（用成熟芯片，专注做好控制交互）
- 不做桌面版（专注移动端场景）
