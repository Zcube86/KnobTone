# 🎛️ KnobTone

> **SmartKnob × HiFi DAC × iPod Nano**
> 一个用物理旋钮 + 力反馈重新定义移动端 HiFi 交互的开源解码耳放

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Status](https://img.shields.io/badge/status-prototype-orange.svg)]()

**KnobTone** 是一款 USB-C 便携解码耳放（小尾巴形态），它的核心不是 DAC 芯片型号，而是一个**带力反馈的物理旋钮**。任何在手机上需要滑动或点击的音乐操作，都可以通过旋转和按压旋钮完成——手指不必触碰屏幕，眼睛不必离开当前在做的事。

---

## ✨ 为什么做这个

手机听歌的体验有一个根本矛盾：**触摸屏不适合盲操作**。调音量要摸到侧边按钮，切歌要掏出手机点亮屏幕，快进要盯着进度条拖——每一步都在打断听歌的状态。

KnobTone 想回答一个问题：

> **如果把触摸播放器的所有操作，重新设计成一套为单旋钮硬件优化的交互语言，会是什么样？**

---

## 🎯 核心特色


| 特色                 | 说明                                                                        |
| -------------------- | --------------------------------------------------------------------------- |
| 🎛️**单旋钮全控制** | 旋转 = 音量 / 切歌 / 快进，按压 = 确认，双击/长按 = 切换模式                |
| 🫳**力反馈触觉**     | 不同模式有不同的阻尼和刻度反馈（音量有 Detent、快进无阻尼、歌单有边界阻力） |
| 🎵**HiFi 音频**      | USB DAC 架构，支持高解析度音频解码                                          |
| 📱**手机即屏幕**     | USB-C 直连手机，手机负责显示和音频源，KnobTone 负责控制和输出               |
| 🔋**内置电池**     | 自带锂电池供电，不消耗手机电量                                              |
| 🔧**完全开源**       | 硬件原理图、PCB、固件、外壳设计全部开源                                     |

## 🏗️ 系统架构

```
┌──────────────────────────────────────────┐
│                 你的手机                   │
│     (USB-C 输出音频 + 接收 HID 指令)        │
└──────────────────┬───────────────────────┘
                   │ USB-C
                   ▼
┌──────────────────────────────────────────┐
│              KnobTone                     │
│                                           │
│  ┌──────────┐   ┌──────────┐   ┌───────┐ │
│  │ 力反馈旋钮 │◄──┤  ESP32   │──►│  DAC  │ │
│  │ BLDC电机  │   │  -S3     │   │ 芯片   │─┼──► 🎧 耳机
│  │ +编码器   │   │          │   │       │ │
│  └──────────┘   │ USB Host │   └───────┘ │
│                  └──────────┘             │
│                       ↑                   │
│                   锂电池                   │
└───────────────────────────────────────────┘
```

**设计原则：模块解耦**

- **音频部分**：由独立 DAC 芯片负责，KnobTone 不处理音频数据
- **控制部分**：ESP32-S3 作为 USB Host 桥接手机，旋钮通过 HID 协议发送媒体控制指令
- **力反馈**：BLDC 无刷电机 + 磁性编码器（SmartKnob 方案），独立于音频链路

## 🎮 交互设计

KnobTone 的交互逻辑借鉴了 [LMusic](https://github.com/nickznd/LMusic) 的设计哲学——把所有高频操作压缩进一个连续的交互流里。


| 操作          | 手机上的等价操作          | KnobTone                         |
| ------------- | ------------------------- | -------------------------------- |
| 调整音量      | 按侧边音量键 / 滑动音量条 | **旋转**（默认模式）             |
| 播放/暂停     | 点击播放按钮              | **按压**                         |
| 上一首/下一首 | 点击切歌按钮              | **双击进入切歌模式 → 旋转**     |
| 快进/快退     | 拖拽进度条                | **长按进入 Seek 模式 → 旋转**   |
| 切换 EQ       | 进入设置菜单              | **三击 → 旋转选择 → 按压确认** |
| 返回          | 滑动手势返回              | **长按**                         |

### 力反馈设计

```
音量模式   │ 旋转每 1% 一个 Detent（咔、咔、咔）
          │ 顶部/底部有软阻力
          │
切歌模式   │ 每首歌一个 Detent
          │ 到歌单顶部/底部：重阻力「咚」
          │
快进模式   │ 完全无刻度，平滑如飞轮
          │ 像 DJ 转盘一样丝滑
```

> 💡 **核心体验**：用户的手在旋钮上就知道自己在哪个模式——不需要看屏幕。

## 🚀 快速开始

> ⚠️ 项目处于早期原型阶段。以下步骤基于当前开发环境。

### 硬件准备


| 组件       | 推荐型号                      | 用途       |
| ---------- | ----------------------------- | ---------- |
| MCU 开发板 | ESP32-S3-DevKitC              | 主控       |
| DAC 模块   | CX31993 USB DAC 小尾巴拆机板  | 音频解码   |
| 旋钮电机   | BLDC 无刷电机 + AS5600 编码器 | 力反馈旋钮 |
| 电机驱动   | TMC6300 / DRV8313             | 电机控制   |
| 电池       | 3.7V 500mAh 锂电池            | 供电       |
| 充电管理   | TP4056 模块                   | 充电       |

### 固件开发

```bash
# 安装 PlatformIO（推荐）
pip install platformio

# 克隆仓库
https://github.com/Zcube86/KnobTone
cd KnobTone/firmware

# 编译 & 烧录
pio run -t upload
```

### MVP 验证（阶段 1）

第一个可运行的版本只需要：

1. ESP32 读取旋钮的旋转方向和按压状态
2. 通过 USB HID 发送媒体控制指令给手机（音量 +/-、播放/暂停）
3. DAC 模块直接透传手机音频

**这个阶段不需要力反馈。**
